# Engineering standards

- TypeScript strict mode and validated boundaries.
- Small modules with one responsibility.
- Stable contracts; vendor behavior stays in adapters.
- Deterministic tests; no network dependency in unit tests.
- Explicit timeouts, cancellation, and bounded retries.
- Atomic checkpoint writes and recoverable state.
- Loopback defaults and least privilege.
- Evidence before completion claims.
- Privacy-safe telemetry and no full prompt logging.
- ADRs for durable architectural decisions.
