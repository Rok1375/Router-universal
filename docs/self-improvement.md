# Human-approved self-improvement

After a run, Router Universal may propose an improvement when evidence suggests repeated failure,
duplicate work, missing capability coverage, excessive context, or a stable reusable pattern.

A proposal includes evidence references, expected benefit, risk, affected files or policies, a test
plan, and an explicit approval state. Proposals are stored separately from runtime configuration.

The router never applies a proposal automatically. An AI coding agent or human implements an approved
change through the normal engineering loop, tests it, and records the decision. Repository-owned
skills and security policies receive additional review.
